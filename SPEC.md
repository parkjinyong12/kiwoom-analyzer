# 키움증권 차트 분석 시스템 — 기획서

**최종 업데이트:** 2026-05-21
**관리:** Claude Code가 git push 전 자동 업데이트

---

## 1. 프로젝트 개요

키움증권 OpenAPI 기반 주식 수급·차트 분석 및 매매신호 알림 시스템.
**주문 실행 없음** — 분석 → 전략 → 리스크 → Slack 알림 파이프라인.

| 항목 | 내용 |
|------|------|
| 배포 | Fly.io (Docker 컨테이너) |
| 언어 | Python 3.11+ |
| 웹 프레임워크 | Flask + Gunicorn |
| DB | PostgreSQL |
| 스케줄링 | APScheduler |
| 알림 | Slack Webhook / slack_sdk |
| 차트 분석 | pandas, pandas-ta, numpy |
| 외부 API | 키움 OpenAPI+ (COM, Windows 전용) |

## 2. 시스템 화면

### 2.1 대시보드
- 전체 매매신호 현황 요약 (매수·관망·매도 비율 파이차트)
- 최근 신호 발생 종목 목록

### 2.2 수급 현황
- 종목별 외국인·기관 수급 추이 차트
- 수급 강도 지표 시각화

### 2.3 수급↑ 가격↔ (Divergence)
- 수급은 증가하나 가격이 횡보 중인 종목 선별 화면

### 2.4 기간별 변화 (Snapshot)
- 특정 기간 기준 수급 변화량 상위 종목 목록

### 2.5 매매 신호
- 전략 에이전트가 생성한 매수·관망·매도 신호 목록
- 신호 필터 (종목, 날짜, 신호 유형)

### 2.6 보유종목 리포트
- 타사 보유종목 가격·수급 변동 일일 리포트 미리보기
- 이메일 수신자 관리 (추가·활성화·삭제)
- 수동 발송 버튼

### 2.7 감사 로그
- 전체 에이전트 이벤트 로그 조회

### 2.8 감시 종목
- 분석 대상 종목 코드·이름 관리 (추가·삭제)

### 2.9 배치 관리
- 5개 배치 작업의 실행 상태 모니터링
- 수동 실행·중지 버튼
- 스케줄 설정 (지정 시간 또는 반복 주기)
- 로그 버튼 클릭 시 별도 팝업 창에서 실시간 로그 확인 (3초 폴링)
- 로그는 실행마다 단일 파일에 append, 실행 구분선 자동 삽입

### 2.10 기획서
- 3탭 구조: [프로젝트 개요] [화면별 기획서] [API 목록]
- 프로젝트 개요: SPEC.md (Claude Code가 push 전 자동 업데이트), marked.js 렌더링
- 화면별 기획서: SCREEN_SPEC.md — 20개 화면별 API 호출·파라미터·응답 문서
- API 목록: Flask url_map 자동 추출, 경로/설명 검색 지원

### 2.11 공통코드 관리
- 시스템 내 공통코드 조회·수정

### 2.12 사용자 관리
- 로그인 계정 목록·권한 관리

### 2.13 보유종목 (외부 연동)
- 타사 증권사 보유종목 수동 입력 및 관리

### 2.14 현재가 관리
- 보유종목 현재가 수동 조회·업데이트

### 2.15 현금성 자산
- 현금·예수금 등 비주식 자산 관리

### 2.16 매크로 관리
- 금리·환율 등 거시 지표 입력 및 조회

### 2.17 리밸런싱 (현금↔주식)
- 현금 비중 목표 대비 현재 비중 분석 및 리밸런싱 제안

### 2.18 신용 관리
- 신용 잔고 및 만기 일정 관리

### 2.19 주식 간 리밸런싱
- 보유 주식 종목 간 비중 재조정 시뮬레이션

### 2.20 테마 리밸런싱
- 테마·섹터 기준 포트폴리오 리밸런싱

## 3. 배치 작업

| 배치 ID | 이름 | 설명 | 로그 파일 |
|---------|------|------|-----------|
| `collect_history` | 수급 히스토리 수집 | 시총 5조 이상 종목 수급 500일치 수집 | `supply_collect.log` |
| `backfill_close` | 종가 백필 | close_price NULL 종목 종가 채우기 | `backfill.log` |
| `holdings_report` | 보유종목 리포트 | 보유종목 가격·수급 변동 리포트 생성 및 이메일 발송 | `holdings_report.log` |
| `sync_prices` | 현재가 동기화 | 타사 보유종목 최신 종가를 키움 API(ka10081)로 조회하여 DB 업데이트 | `sync_prices.log` |
| `run_once` | 매매신호 갱신 | 전 감시종목 파이프라인 1회 실행 (수급 수집 + 차트 분석 + 매매신호 생성) | `run_once.log` |

**배치 로그 정책:**
- 실행마다 새 파일 대신 `{prefix}.log` 단일 파일에 누적 append
- 각 실행 시작 시 구분선(`=== 새 실행 시작 · YYYY-MM-DD HH:MM:SS ===`) 자동 삽입
- 기존 타임스탬프 파일 있으면 하위 호환 유지

**스케줄링 방식:**
- 지정 시간 (요일·시·분 지정 — Cron 방식)
- 반복 주기 (N분마다, 선택적 허용 시간대 지정)

| Audit/Monitor | `agents/audit_monitor.py` | 전체 이벤트 로깅, 시스템 상태 감시 |

## 5. 인프라 및 배포

- **플랫폼:** Fly.io (단일 컨테이너)
- **CI/CD:** GitHub Actions — `main` 브랜치 push 시 자동 배포
- **배포 알림:** 성공/실패 시 Slack 메시지 발송
- **DB:** 외부 PostgreSQL (Fly.io Postgres 또는 별도 호스팅)
- **환경변수:** `DATABASE_URL`, `SLACK_WEBHOOK_URL` 등 Fly.io secrets 관리
